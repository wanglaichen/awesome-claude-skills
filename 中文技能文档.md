# Awesome Claude Skills 中文文档

## 仓库介绍

这是一个精心策划的实用 Claude Skills 列表，旨在提高 Claude.ai、Claude Code 和 Claude API 的生产力。

> **想要超越生成文本的技能？** Claude 可以发送电子邮件、创建问题、发布到 Slack，并在 1000+ 个应用程序中执行操作。[了解更多 →](./connect/)

## Claude Skills 是什么？

Claude Skills 是可定制的工作流程，用于教导 Claude 如何根据您的独特需求执行特定任务。技能使 Claude 能够在所有 Claude 平台上以可重复、标准化的方式执行任务。

## 技能分类

### 文档处理
- **docx** - 全面的文档创建、编辑和分析工具，支持跟踪更改、评论、格式保留和文本提取。适用于创建新文档、修改或编辑内容、处理跟踪更改、添加评论等任何文档任务。
  **使用示例**：
  - 创建新的 Word 文档并添加格式化内容
  - 编辑现有文档并添加跟踪更改
  - 分析文档内容并提取关键信息
  **注意事项**：
  - 对于法律、学术、商业或政府文档，建议使用"Redlining workflow"以确保更改可追踪
  - 批量处理时建议将相关更改分组为 3-10 个更改的批次，以提高效率和可调试性

- **pdf** - 全面的 PDF 操作工具包，用于提取文本和表格、创建新 PDF、合并/拆分文档以及处理表单。适用于填写 PDF 表单或以编程方式大规模处理、生成或分析 PDF 文档。
  **使用示例**：
  - 从 PDF 中提取文本和表格数据
  - 合并多个 PDF 文件为一个文档
  - 分割大型 PDF 文件为多个较小的文件
  **注意事项**：
  - 对于扫描的 PDF，需要使用 OCR 工具进行文本提取
  - 处理加密 PDF 时需要提供密码

- **pptx** - 读取、生成和调整幻灯片、布局和模板的工具。适用于创建新的演示文稿、修改现有幻灯片、调整布局和格式等任务。
  **使用示例**：
  - 创建新的 PowerPoint 演示文稿
  - 修改现有演示文稿的布局和内容
  - 为演示文稿添加图表和图形

- **xlsx** - 电子表格操作工具，支持公式、图表和数据转换。适用于数据分析、报表生成、数据可视化等任务。
  **使用示例**：
  - 创建包含公式和函数的电子表格
  - 生成数据可视化图表
  - 执行数据转换和分析

### 开发和代码工具
- **artifacts-builder** - 使用现代前端网络技术（React、Tailwind CSS、shadcn/ui）创建复杂的多组件 claude.ai HTML 构件的工具套件。适用于创建视觉效果丰富、交互性强的网页构件。
- **Changelog Generator** - 通过分析历史记录并将技术提交转换为客户友好的发布说明，自动创建面向用户的变更日志。适用于项目发布管理和版本控制。
- **LangSmith Fetch** - 通过自动从 LangSmith Studio 获取和分析执行跟踪来调试 LangChain 和 LangGraph 代理。第一个用于 Claude Code 的 AI 可观察性技能。适用于 AI 代理开发和调试。
- **MCP Builder** - 指导创建高质量的 MCP（模型上下文协议）服务器，用于使用 Python 或 TypeScript 将外部 API 和服务与 LLM 集成。适用于构建自定义 API 集成和扩展 LLM 功能。
- **Skill Creator** - 提供创建有效 Claude Skills 的指导，这些技能通过专业知识、工作流程和工具集成扩展功能。适用于希望创建自定义技能的开发者。
  **使用示例**：
  - 初始化新技能：`scripts/init_skill.py <skill-name> --path <output-directory>`
  - 编辑 SKILL.md 文件，添加技能描述、使用场景和操作说明
  - 打包技能：`scripts/package_skill.py <path/to/skill-folder>`
  **注意事项**：
  - 使用命令式/不定式形式（动词优先指令）编写技能，而不是第二人称
  - 将详细参考材料、架构和示例移至参考文件，保持 SKILL.md 精简
  - 确保技能包含明确的用例和边界情况
- **Connect** - 将 Claude 连接到任何应用程序。发送电子邮件、创建问题、发布消息、更新数据库 - 在 Gmail、Slack、GitHub、Notion 和 1000+ 服务中执行实际操作。适用于跨应用程序自动化和集成。
- **Webapp Testing** - 使用 Playwright 测试本地 Web 应用程序，验证前端功能，调试 UI 行为，并捕获屏幕截图。适用于 Web 应用程序开发和测试。

## 快速开始：将 Claude 连接到 500+ 应用

**connect-apps** 插件让 Claude 执行实际操作 - 发送电子邮件、创建问题、发布到 Slack。它处理身份验证并通过 Composio 连接到 500+ 应用。

### 1. 安装插件

```bash
claude --plugin-dir ./connect-apps-plugin
```

### 2. 运行设置

```
/connect-apps:setup
```

在提示时粘贴您的 API 密钥。（在 [platform.composio.dev](https://platform.composio.dev/?utm_source=Github&utm_content=AwesomeSkills) 获取免费密钥）

## 如何使用技能

### 在 Claude.ai 中使用技能

1. 点击聊天界面中的技能图标 (🧩)
2. 从市场添加技能或上传自定义技能
3. Claude 会根据您的任务自动激活相关技能

### 在 Claude Code 中使用技能

1. 将技能放在 `~/.config/claude-code/skills/` 中：
   ```bash
   mkdir -p ~/.config/claude-code/skills/
   cp -r skill-name ~/.config/claude-code/skills/
   ```

2. 验证技能元数据：
   ```bash
   head ~/.config/claude-code/skills/skill-name/SKILL.md
   ```

3. 启动 Claude Code：
   ```bash
   claude
   ```

4. 技能会自动加载并在相关时激活

### 通过 API 使用技能

使用 Claude Skills API 以编程方式加载和管理技能：

```python
import anthropic

client = anthropic.Anthropic(api_key="your-api-key")

response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    skills=["skill-id-here"],
    messages=[{"role": "user", "content": "Your prompt"}]
)
```

有关详细信息，请参阅 [Skills API 文档](https://docs.claude.com/en/api/skills-guide)。

## 如何创建技能

### 技能结构

每个技能都是一个包含 `SKILL.md` 文件的文件夹，带有 YAML 前置内容：

```
skill-name/
├── SKILL.md          # 必需：技能说明和元数据
├── scripts/          # 可选：辅助脚本
├── templates/        # 可选：文档模板
└── resources/        # 可选：参考文件
```

### 基本技能模板

```markdown
---
name: my-skill-name
description: A clear description of what this skill does and when to use it.
---

# My Skill Name

Detailed description of the skill's purpose and capabilities.

## When to Use This Skill

- Use case 1
- Use case 2
- Use case 3

## Instructions

[Detailed instructions for Claude on how to execute this skill]

## Examples

[Real-world examples showing the skill in action]
```

### 技能最佳实践

- 专注于特定、可重复的任务
- 包含清晰的示例和边缘情况
- 为 Claude 编写说明，而不是最终用户
- 在 Claude.ai、Claude Code 和 API 中测试
- 记录先决条件和依赖项
- 包含错误处理指导
